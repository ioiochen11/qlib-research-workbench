# 路线图

## 近期

- 在当前最小加权聚合之外，继续改进模型筛选逻辑
- 为 `train -> report -> review -> backtest` 增加更多 smoke 覆盖
- 优化打包方式、命令体验，以及首次运行配置
- 让输出目录和清理操作更容易管理

## 中期

- 支持更灵活的报表过滤规则和评分规则
- 增加更丰富的实验对比汇总表
- 更清晰地拆分研究逻辑和展示层输出
- 增加面向 dashboard 或静态页面的可选导出接口

## 更长期

- 为模型定义和选股规则提供更清晰的插件式布局
- 增加更可复现的环境与发布工具链
- 明确项目未来是继续做研究工作台，还是往更广的平台方向演进
